=head0 *Предговор

Книгата I<Модерен Perl> е начин да се опише как работят най-ефективните в света 
програмисти на Perl 5. Те използват идиоми. Те се възползват от CPAN.
Те показват добър вкус и умения да пишат мощен, лесен за поддръжка, мащабируем,
стегнат и ефективен код. Вие също можете да придобиете тези умения.

Perl се появява за първи път през 1987 като просто средство за системна администрация. 
Въпреки, че отначало заема удобната ниша между писането на шел-скриптове и програмирането на C, 
по-късно се превръща в мощен език за програмиране с общо предназначение.
Историята на Perl 5 е изпълнена с прагматизъм, a бъдещето му с подобрения и блясъкN<Perl 6 е преоткриване на програмирането, основано на здравите принципи на 
Perl, но е предмет на друга книга.>. 

Историят на езика е дълга. Разбирането за писане на велики програми на Perl се промени значително за последните 17 години. Може да пишете програми с добра производителност без да се възползвате от пълната функционалност на езика. Но програмистите от световната общност на Perl са изобретили, заели, подобрили и шлифовали идеи и умения, които са предоставили свободно на всеки, който иска да ги усвои.

=head1 Използване на Модерен Perl

Модулът C<Modern::Perl> от CPAN (L<cpan>) ви предупреждава при писане на съмнителни конструкции и при допуснати грешки от недоглеждане. Също така включва новите възможности, въведени в модерните издания на Perl 5. Всички примери спрограмен код в книгата предполагат, че програмата ви започва по следния начин:

=begin programlisting

    #!/usr/bin/env perl

    use Modern::Perl 2011;
    use autodie;

=end programlisting

... which is equivalent to:

=begin programlisting

    #!/usr/bin/env perl

    use 5.012;      # включва "use strict;"
    use warnings;
    use autodie;

=end programlisting

Някои примери използват тестови функции от рода на C<ok()>, C<like()> и C<is()>
(L<testing>). Тези програми следват следния шаблон:

=begin programlisting

    #!/usr/bin/env perl

    use Modern::Perl;
    B<use Test::More;>

    # примерният код е тук

    B<done_testing();>

=end programlisting

At the time of writing, the current stable Perl 5 release family is Perl 5.14.
The examples in this book work best with Perl 5.12.0 or newer. Many examples
will work on older versions of Perl 5 with modest changes, but you will have
more difficulty with anything older than 5.10.0.

X<Strawberry Perl>
X<ActivePerl>
X<perlbrew>
X<CPAN; C<App::perlbrew>>

If you have no Perl 5 installed (or if you have an old version installed), you
can install a newer release yourself. Windows users, download Strawberry Perl
from U<http://www.strawberryperl.com/> or ActivePerl from
U<http://www.activestate.com/activeperl>. Users of other operating systems with
Perl 5 already installed (and a C compiler and the other development tools),
start by installing the CPAN module C<App::perlbrew>N<See
U<http://search.cpan.org/perldoc?App::perlbrew> for installation
instructions.>.

C<perlbrew> allows you to install and manage multiple versions of Perl 5. This
allows you to switch between versions of Perl 5 as well as to install Perl 5
and CPAN modules in your home directory without affecting the system's version.
If you've ever had to beg your system administrator for permission to install
software, you know how much easier your life can be now.
